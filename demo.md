# 演示脚本 🎬

用于展示 AI Prompt 管理器的核心功能。

## 🎯 演示流程（5 分钟）

### 1. 介绍（30 秒）

**开场白**：
> 大家好！今天给大家展示一个我刚完成的项目 —— **AI Prompt 管理器**。
> 
> 这是一个专门为 AI 绘图创作者设计的 Chrome 浏览器扩展，可以帮助你高效地管理、组织和复用你的 Prompt。

**展示要点**：
- 这是一个浏览器扩展（展示工具栏图标）
- 支持 Midjourney、Stable Diffusion 等主流 AI 绘图工具
- 完全本地存储，保护隐私

---

### 2. 核心功能演示（3 分钟）

#### 功能 1: 右键保存 Prompt（30 秒）

**操作步骤**：
1. 打开一个包含 Prompt 的网页（如 Midjourney 社区）
2. 选中一段 Prompt 文本：
   ```
   masterpiece, best quality, 1girl, cyberpunk city, neon lights, night scene
   ```
3. 右键点击 → 选择「**保存为 Prompt**」
4. 看到通知：「Prompt 已保存」

**讲解要点**：
> 只需要右键点击，就能快速保存看到的优质 Prompt。不需要复制粘贴到记事本了。

---

#### 功能 2: 侧边栏管理（1 分钟）

**操作步骤**：
1. 按快捷键 `Ctrl+Shift+P`（或点击扩展图标）
2. 侧边栏从右侧滑出
3. 展示界面：
   - 顶部：搜索框 + 排序按钮
   - 中间：Prompt 卡片列表
   - 底部：统计信息

**讲解要点**：
> 所有保存的 Prompt 都在这里。界面简洁美观，一目了然。

---

#### 功能 3: 标签系统（1 分钟）

**操作步骤**：
1. 点击「赛博朋克少女」这个示例 Prompt
2. 指出不同颜色的标签：
   - 🔵 蓝色 = 主体（1girl）
   - 🟣 紫色 = 风格（cyberpunk）
   - 🟡 黄色 = 光效（neon lights）
   - 🩷 粉色 = 技法（masterpiece）
3. 指出标签旁的数字（权重）

**讲解要点**：
> 不是简单的文本存储，而是结构化的标签系统。
> 每个标签有类型、权重，方便管理和调整。

---

#### 功能 4: 一键复制（30 秒）

**操作步骤**：
1. 找到任意 Prompt 卡片
2. 点击右下角的「复制」图标
3. 图标变绿，提示已复制
4. 打开一个文本框，粘贴展示

**讲解要点**：
> 一键复制，自动转换为 Midjourney 或 Stable Diffusion 格式。
> 包括所有参数和权重。

---

#### 功能 5: 搜索和过滤（30 秒）

**操作步骤**：
1. 在搜索框输入「赛博朋克」
2. 列表实时过滤
3. 点击「收藏」按钮
4. 只显示收藏的 Prompt

**讲解要点**：
> 支持全文搜索，包括标题、描述、标签。
> 可以按时间、收藏、标题排序。

---

### 3. 技术亮点（1 分钟）

**快速介绍技术栈**：
- React 18 + TypeScript（类型安全）
- Vite 5（快速构建）
- Tailwind CSS（现代化 UI）
- Chrome Manifest V3（最新标准）

**强调差异化**：
1. **标签碎片化设计**
   - 不是简单的字符串，而是结构化数据
   - 每个标签可以独立编辑、调整权重
   
2. **多格式支持**
   - 同一个 Prompt 可以导出为不同格式
   - 自动处理语法差异

3. **监控系统思维**
   - 数据模型预留了版本追踪
   - 为未来的效果评估做准备

## 💬 常见问题准备

### Q1: 数据存储在哪里？
**A**: 完全存储在浏览器本地（Chrome Storage API），不上传到云端，保护隐私。

### Q2: 支持哪些 AI 工具？
**A**: 目前支持 Midjourney 和 Stable Diffusion 格式。未来会支持更多。

### Q3: 可以导出数据吗？
**A**: 这个功能在第四周的计划中，会支持 JSON 格式导出/导入。

### Q4: 是否开源？
**A**: 是的，完全开源，欢迎贡献代码和提建议。

### Q5: 会上架 Chrome Web Store 吗？
**A**: 计划在完成前三周功能后上架，预计一个月内。

### Q6: 支持移动端吗？
**A**: 目前只支持桌面浏览器。移动端可以考虑开发独立的 App。

### Q7: 如何参与开发？
**A**: 查看 GitHub 仓库的 DEVELOPMENT.md，欢迎提 Issue 和 PR。
